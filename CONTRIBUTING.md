# Contributing

Haunted Studio values falsifiable research and reliable software over claims of
machine consciousness or artistic personhood.

Before opening a pull request:

1. explain the behavior or research property being changed;
2. identify the strongest alternative explanation;
3. add or update isolated tests;
4. preserve ledger and intention-lock ordering;
5. document scoring, threshold, condition, or constitution changes;
6. confirm observation, reference, image, and review rights;
7. run `npm ci`, `npm run validate`, `npm run demo`, and
   `npm run experiment:smoke`; and
8. ensure generated runtime data and credentials are not staged.

Negative or inconclusive findings are useful. Do not hide them.

By participating, you agree to follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
